# PLC 通信库诊断指南

## 问题诊断流程

### 1. 确认 PLC 模拟器已启动

| 协议 | 推荐模拟器 | 默认端口 |
|------|-----------|---------|
| 西门子 S7 | PLCSIM Advanced / Snap7 Server | 102 |
| 三菱 MC | GX Works2 / MX Component | 5006 |
| 欧姆龙 FINS | CX-Programmer / FinsGateway | 9600 |
| Modbus TCP | ModRSsim / Modbus Slave | 502 |

### 2. 西门子 S7 常见错误

#### 错误码 0x04 - 地址无效
**原因**：DB 块不存在或地址格式错误
**解决方案**：
1. 使用 M 区测试：`MW0`
2. 在 PLC 中创建 DB1 数据块
3. 确认 DB 块属性为"非优化"/"标准访问"

#### 错误码 0x05 - 地址超出范围
**原因**：写入地址不存在或只读
**解决方案**：
1. 确认地址可写入
2. 使用 MW0 测试写入

#### 错误码 0x03 - 访问被拒绝
**原因**：PLC 保护级别或 DB 块保护
**解决方案**：
1. 降低 PLC 保护级别
2. 取消 DB 块的"仅系统访问"属性

### 3. 三菱 MC 常见错误

#### 错误码 0xC300 - 设备不存在
**原因**：地址超出 PLC 范围
**解决方案**：
- D 区：D0-D7999（Q 系列）
- M 区：M0-M7679
- X/Y 区：使用 16 进制（X0, X10, X20）

### 4. 欧姆龙 FINS 常见错误

#### 连接超时
**原因**：FINS/TCP 服务未启动
**解决方案**：
1. 确认 CX-Programmer 可连接
2. 检查端口 9600 是否开放
3. 确认 FinsGateway 已安装

### 5. Modbus TCP 常见错误

#### 异常码 0x02 - 非法数据地址
**原因**：寄存器地址不存在
**解决方案**：
- 保持寄存器：40001-49999
- 输入寄存器：30001-39999
- 线圈：00001-09999

## 测试步骤

### 西门子 S7 测试

1. **启动模拟器**（PLCSIM Advanced 或 Snap7 Server）
2. **设置 IP**：127.0.0.1（本地测试）
3. **设置端口**：102
4. **选择 PLC 类型**：S1200 或 S1500
5. **测试地址**：
   - ✅ `MW0` - 内部存储区（推荐）
   - ✅ `IW0` - 输入区
   - ✅ `QW0` - 输出区
   - ⚠️ `DB1.DBW0` - 需要先创建 DB1

### 三菱 MC 测试

1. **启动模拟器**（GX Works2 仿真）
2. **设置 IP**：127.0.0.1
3. **设置端口**：5006
4. **测试地址**：
   - ✅ `D0` - 数据寄存器
   - ✅ `M0` - 内部继电器
   - ✅ `X0` - 输入（16进制）
   - ✅ `Y0` - 输出（16进制）

### Modbus TCP 测试

1. **启动模拟器**（ModRSsim 或 Modbus Slave）
2. **设置 IP**：127.0.0.1
3. **设置端口**：502
4. **设置从站 ID**：1
5. **测试地址**：
   - ✅ `0` - 寄存器偏移
   - ✅ `40001` - 保持寄存器

## 日志分析

### 正常的 S7 读取流程

```
[Send] 37 bytes: 03 00 00 25 02 F0 80 32 01 00 00 00 ...
[Recv] 响应正确，返回码 0xFF
```

### 异常响应

```
[Warning] Protocol error: S7 read error: return code 0x04
```

**排查步骤**：
1. 检查地址格式是否正确
2. 确认 PLC 中该地址存在
3. 尝试其他地址（如 MW0）

使用 UI 界面的"日志输出"Tab 查看：

- `[Warning]` - 协议错误

## 常见问题 FAQ

### Q: 为什么读取 DB1.DBW0 报错？
A: DB1 需要在 PLC 中手动创建。建议先测试 MW0。

### Q: 为什么连接超时？
A: 
1. 检查 PLC 模拟器是否启动
2. 检查端口是否正确
3. 检查防火墙设置

### Q: 为什么三菱读取返回空数据？
A: 
1. 检查模拟器是否处于 RUN 模式
2. 检查地址格式（X/Y 使用 16 进制）

### Q: 如何确认协议实现正确？
A: 查看 `[Send]` 日志，对比协议规范文档。
